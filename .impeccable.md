## Design Context

### Users
- Smart glasses wearers using Rokid Q95 Pro in daily life scenarios
- Context: walking, commuting, working, exercising — always on the move
- Job to be done: access information, navigate, communicate, and control devices hands-free through an AR overlay that augments rather than replaces their real-world view
- Technical users and early adopters comfortable with cutting-edge wearable tech

### Brand Personality
- **Futuristic** · **Tech-forward** · **Ultimate precision**
- Voice: confident, minimal, forward-looking — speaks through design rather than words
- Tone: calm authority with a sense of wonder; never loud, never boring

### Aesthetic Direction
- **Visual tone**: Dark-mode-first AR HUD aesthetic with depth and dimensionality — not flat, not skeuomorphic, but layered glass-like surfaces with subtle luminosity
- **References**:
  - Xiaomi SU7 HUD interface — clean data hierarchy, glowing accents on dark canvas, information density without clutter
  - Apple Vision Pro — spatial UI language, frosted glass materials, precise typography, ambient light response
- **Anti-references**:
  - No skeuomorphism or heavy textures
  - No flashy/gaudy decorations or gratuitous gradients
  - No flat/lifeless 2D layouts — everything should feel like it exists in 3D space with depth cues
  - No generic dashboard/admin panel aesthetics
- **Theme**: Exclusively dark mode — deep navy-black (#0a0a1a → #1a1a2e) with luminous blue (#64C8FF) as primary accent
- **Materials**: Semi-transparent frosted panels, subtle glow effects, glass-morphism with restraint

### Emotional Goals
- **Immersion** without isolation — AR content feels part of the real world, not blocking it
- **Tech-forward delight** — every gaze interaction, every icon hover should deliver a satisfying sci-fi feedback loop
- **Effortless control** — complex multi-modal interactions (voice, gesture, gaze, touch) feel intuitive and unified

### Design Principles

1. **Depth over flatness** — Use layered transparency, subtle shadows, and glow effects to create spatial hierarchy. UI elements should feel like they float at different distances in the user's field of view.

2. **Information at a glance** — AR displays demand instant comprehension. Use strong typographic hierarchy, strategic color coding, and minimal text. Every pixel must earn its place on a 40° FOV display.

3. **Responsive luminosity** — Interactive elements respond with light: glow on focus, pulse on activation, ripple on confirmation. The UI breathes and reacts like a living system, not a static screen.

4. **Restrained sci-fi** — Futuristic doesn't mean busy. Channel the precision of aerospace HUDs: clean lines, purposeful animation, monospaced data readouts where appropriate. Every decorative element must serve a functional purpose.

5. **Multi-modal coherence** — Whether triggered by gaze, voice, gesture, or touch, the visual feedback language stays consistent. Same easing curves, same color semantics, same spatial logic across all input modes.

### Technical Constraints
- **Display**: 800×600 demo viewport simulating 1280×720 single-eye AR display, 40° FOV
- **Layout**: Three-zone system — Smart Task Zone (top-left), Status Bar (top-right), Main Task Area (center)
- **Color palette**: Primary blue `rgba(100, 200, 255, 0.9)`, secondary purple `rgba(180, 130, 255, 0.85)`, accent amber `rgba(255, 180, 60, 0.9)`, base background `#0a0a1a`
- **Typography**: System sans-serif stack, sizes 10–32px, weights 400–700
- **Spacing**: 4px base grid system (2/4/8/12/16/24/32/48)
- **Animation**: Fast 150ms (micro), Normal 250ms (state), Slow 400ms (layout). Easing: cubic-bezier curves, no bounce/elastic
- **Interaction**: 6 input modes — side touchpad, voice, EMG band, camera gesture, head tracking, physical button
- **Stack**: React 19, TypeScript 5.9, Vite 8, Three.js (for 3D/dual-eye rendering)
