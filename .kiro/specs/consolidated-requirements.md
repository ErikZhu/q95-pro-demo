# Q95 Pro 智能眼镜 — 总需求文档

> 汇总自 `q95-pro-pre-research` 和 `orb-menu-hub` 两个分需求，仅保留开发与设计必要参考信息。

---

## 1. 硬件约束

| 参数 | 规格 |
|------|------|
| 显示方案 | Micro-OLED（主推）/ MicroLED（备选） |
| 单眼分辨率 | ≥1280×720（Micro-OLED 可达 1920×1080） |
| 峰值亮度 | ≥3000nits |
| 刷新率 | ≥60Hz |
| FOV | ≥40° |
| 操作系统 | Android + Vela 魔改 |
| 交互方式 | 侧边触控、语音、EMG 手势、摄像头手势、头部追踪、物理按键（共 6 种） |
| 跨设备通信 | BLE 5.0 + Wi-Fi Direct |
| Demo 技术栈 | Web（Vite + React + TypeScript），视口 800×600（⚠️ 仅 Demo） |

## 2. 屏幕布局

三分区常驻布局：

| 分区 | 位置 | 内容 |
|------|------|------|
| Smart_Task_Zone | 左上角 | AI 状态球（AI_Status_Orb）+ 任务摘要 + Orb 菜单入口 |
| Status_Bar | 右上角 | 电量、蓝牙、Wi-Fi、时间 |
| Main_Task_Area | 中央 | 当前应用内容 |

- 三分区始终可见，不互相遮挡
- Main_Task_Area 占据主要空间

## 3. AI 状态球与 Orb 菜单（核心交互）

### 3.1 AI_Status_Orb 状态

| 状态 | 视觉表现 |
|------|----------|
| idle | 静态柔光 |
| listening | 脉冲呼吸 |
| thinking | 旋转动效 |
| responding | 扩散波纹 |

### 3.2 Orb 菜单状态机

状态：`orb_idle` → `orb_hint` → `orb_menu_open` → `orb_item_focused`

| 转换 | 触发条件 |
|------|----------|
| idle → hint | 注视 Orb >0.8s |
| idle → menu_open | 触控单击 / EMG 捏合 |
| hint → menu_open | 继续注视 >0.3s / 触控 / EMG |
| hint → idle | 视线移开 |
| menu_open → item_focused | 注视菜单项 >0.5s |
| menu_open → idle | 视线移开 >2s / 摇头 / 侧滑 |
| item_focused → idle | 确认选择（点头/EMG/触控）→ 启动应用 |
| item_focused → menu_open | 视线移开菜单项 |

### 3.3 菜单项（当前 4 项）

| ID | 图标 | 标签 | 路由 |
|----|------|------|------|
| notifications | bell | 通知 | notification_center |
| camera | camera | 相机 | camera |
| messaging | chat | 消息 | messaging |
| settings | gear | 设置 | settings |

- 360° 环形均匀分布，围绕 AI_Status_Orb
- Orb 位于视口中心 (400, 300)，半径 80px
- 菜单项尺寸：圆 42px，图标 18px，字号 9px

### 3.4 与 Smart_Task_Zone 互斥

- Smart_Task_Zone expanded 时，Orb 菜单禁止展开
- Orb 菜单展开时，Smart_Task_Zone 禁止进入 confirm_prompt / expanded
- 菜单收起后，Smart_Task_Zone 恢复正常

### 3.5 动画规格

| 动画 | 时长 | 缓动 | 备注 |
|------|------|------|------|
| 菜单展开 | 300ms | cubic-bezier(0.34, 1.56, 0.64, 1) | 菜单项依次弹出，间隔 30ms |
| 菜单收起 | 250ms | — | 从外到内依次收回，间隔 20ms |
| 菜单项高亮 | 200ms | — | 边框发光 + 图标放大 |
| 目标帧率 | 60fps | — | — |

## 4. Smart_Task_Zone 详细行为

- 紧凑模式：显示 AI_Status_Orb + 当前任务摘要文本
- 注视 >1s → 显示"是否展开？"确认提示
- 点头 / EMG 捏合双击 → 300ms 内展开详细任务视图
- 视线移开 >3s / 摇头 / 侧滑 → 自动收回
- 支持多任务轮播

## 5. 状态栏（Status_Bar）

- 常驻显示：电量%、蓝牙、Wi-Fi、时间
- 电量 <20%：警告色；<5%：全屏警告
- 连接变化 500ms 内更新图标
- 支持注视/点击展开详情

## 6. Launcher（主界面）

- 开机 3s 内加载完成
- 默认视野清爽，中央无 UI 元素
- ≤6 个常用应用收纳在收纳图标中，注视 >0.8s 展开
- 视线移开 >2s 自动收起

## 7. 多模态交互

### 优先级（高→低）

1. 物理按键 — 紧急操作
2. 语音 — 自然交互
3. EMG 手势 — 精确手势
4. 侧边触控 — 常规操作
5. 摄像头手势 — 视觉手势
6. 头部追踪 — 光标控制

- 多输入同时触发时按优先级解决冲突
- 响应延迟 <100ms
- 传感器不可用时自动切换并通知用户

## 8. AI 语音助手

- 唤醒词激活 <500ms
- 语音指令响应 <2s
- 支持自然语言、模糊指令、上下文理解
- 5s 无输入自动退出
- 低置信度时请求确认

## 9. 任务中心（跨设备控制）

通过 AI 语音助手统一控制：

| 目标 | 能力 |
|------|------|
| 眼镜本机 | 音量、拍照、亮度、夜间模式、省电、Wi-Fi |
| 配对手机 | 媒体播放、日程、待办、拨打电话 |
| IoT 设备 | 状态查询、场景自动化、单设备控制 |
| 车机 | 车辆状态、导航发送、车窗/空调控制 |
| 手表 | 步数查询、运动记录 |
| 三方应用 | 社交/娱乐/办公类语音控制 |

- 支持多步骤复合指令自动拆解
- 设备不可达时提示并建议替代操作
- 执行状态实时显示在 Smart_Task_Zone

## 10. 功能模块摘要

### 通知中心
- 新通知在视野边缘提示
- 时间倒序，支持按应用分组
- 侧边触控下滑展开列表
- >50 条自动归档已读

### AR 导航
- 3s 内规划路线
- 视野叠加方向箭头、距离、ETA
- 偏离 2s 内重新规划
- GPS 丢失时用最近位置继续

### 拍照/录像
- 物理按键/手势触发，300ms 内完成拍照
- ≥1080p，录制时显示时长和剩余空间
- 自动同步到手机相册

### 音乐播放
- 播放/暂停/切歌/音量
- 侧边触控双击切换播放状态
- 来电自动暂停，通话后恢复

### 实时翻译
- 检测外语 1s 内显示翻译
- ≥10 种语言，支持语音和 OCR 输入
- 离线模式降级可用

### 提词器
- 可配置字号，自动滚动
- 侧边触控调速，手势暂停
- 半透明显示，不遮挡环境

### 健康监测
- 步数、心率、卡路里
- 自动识别运动类型
- 心率异常即时警告

### 消息/通话
- 新消息视野内预览
- 语音回复，来电接听/拒绝
- 蓝牙断开提示重连

### 设置面板
- 分类：显示、声音、连接、交互、隐私、系统
- 修改即时生效 + 视觉确认

### 手机信息流转（Phone Relay）
- 支持类型：外卖、日程、来电、机票、打车、电影票、微信、音乐
- 2s 内推送到眼镜，差异化 Info_Card 模板
- 按时效性 + 优先级智能排序
- 侧边触控滑动浏览，点击展开详情
- 蓝牙断开保留最近同步数据

### 应用商店
- 通过配对手机浏览/下载
- 自动推送安装，空间不足提示

## 11. 设计规范输出要求

- 统一视觉语言：颜色体系、字体、图标风格、动效
- 手势映射表
- 布局规范：安全区域、信息层级、视觉焦点
- 导航流程与页面跳转逻辑
- 核心界面 UI 原型
- 技术可行性分析报告

## 12. Demo 专属需求（非产品需求）

> ⚠️ 以下需求仅用于 Web Demo 演示目的，不属于产品核心功能。实际产品中不包含这些功能。

### 12.1 交互模拟面板（DemoControlPanel）

- 右侧滑出式浮动面板，模拟 6 种多模态交互输入
- 物理按键：电源/确认/返回/音量，支持键盘快捷键映射
- 语音：文本输入框模拟语音指令
- EMG 手环：捏合/双击捏合/握拳按钮
- 侧边触控：上下左右滑动 + 点击，支持方向键映射
- 摄像头手势：手掌停止/指向/挥手/点赞按钮
- 头部追踪：点头/摇头按钮 + 模拟眼动区域（鼠标 hover 模拟注视光标）
- 模拟眼动光标：主视口中显示 14px 蓝色光标球，跟随模拟注视位置
- 光标碰撞检测：光标移到 AI 语音球上可触发菜单唤起，移到菜单项上可触发聚焦高亮

### 12.2 单目/双目切换

- 底部按钮切换单目模式（单 lens 全宽）和双目模式（左右 lens 带视差偏移）
- 双目模式下左右各标注 L/R 标签
- 仅用于演示 AR 眼镜双目显示效果，实际产品由硬件决定

### 12.3 Demo 通用要求

- 可运行交互式 Demo，覆盖 Launcher + 通知中心 + ≥3 个核心模块
- 接近真实的动效和响应速度
- 支持 PC/手机浏览器运行
- 技术栈：Vite + React + TypeScript，视口 800×600
